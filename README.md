# インタプリタ

ソースコード -> トークン列 -> 抽象構文木

## 字句解析

ソースコードをトークン列に変換する。

- トークン: 言語の最小単位 (キーワード、識別子、リテラル、演算子など)
- 字句解析器 (Lexer): ソースコードを読み込み、トークン列を生成するプログラム

## 構文解析

トークン列を抽象構文木 (AST) に変換する。

- 抽象構文木 (AST): プログラムの構造を表す木構造
- 構文解析器 (Parser): トークン列を読み込み、ASTを生成するプログラム

## REPL

Read-Eval-Print Loop の略。対話型のプログラム実行環境。

- Read: ユーザーからの入力を読み込む
- Eval: 入力を評価 (実行) する
- Print: 評価結果を表示する
- Loop: これらのステップを繰り返す

## 構文解析器（Parser）

構文解析器は、入力をあるデータ構造へと変換する。それも入力を反映したデータ構造である。構文解析器は、字句解析器からのトークン列を受け取り、抽象構文木 (
AST) を生成する。

- トップダウン構文解析: 入力の先頭から順に解析を進める方法。再帰下降構文解析が代表的。
- ボトムアップ構文解析: 入力の末尾から順に解析を進める方法。LR構文解析が代表的。

ステートメントと式

- ステートメント: プログラムの命令を表す。例: 変数宣言、関数定義、制御構造 (if文、ループなど)
- 式: 値を生成する構造。例: 算術演算、関数呼び出し、リテラル

1.let文の解析
`let x = 5;`
この文は「let文」と呼ばれ、与えられた名前を値に束縛する。

`let <identifier> = <expression>;`

- `<identifier>`: 識別子
- `<expression>`: 式

式と文の違い

- 式 (Expression): 値を生成するコードの断片。例: `5 + 3`, `foo(10)`
- 文 (Statement): プログラムの命令を表すコードの断片。
- 式は値を生成し、文は生成しない。
